# QinCode 打包分发 — 变更文档

## 第2次变更
- **时间**: 2026-06-22 12:17
- **对话次数**: 2

### 变更内容

#### 1. 修复 SEA wrapper HTML 文件路径问题

**问题**: SEA wrapper 将 HTML 资源提取到 `/tmp/qincode-sea/`，但 `oauth.ts` 中硬编码了 `join(__dirname, '../../login-callback.html')`，在 SEA 环境中路径不正确。

**修复**:
- `scripts/bundle.mjs`: SEA wrapper 在提取 HTML 文件后设置 `process.env.QINCODE_HTML_DIR = tmpDir`
- `src/auth/oauth.ts`: 优先从 `process.env.QINCODE_HTML_DIR` 读取路径，回退到原 `__dirname` 方式

#### 2. 修复 install.sh macOS Gatekeeper 问题

**问题**: macOS 通过 curl 下载的二进制可能被系统标记 quarantine 属性，Gatekeeper 可能阻止运行。

**修复**:
- `scripts/install.sh`: 安装后自动执行 `xattr -d com.apple.quarantine` 移除 quarantine 属性
- `cloudflare-site/install.sh`: 同步更新

#### 3. 修复 --version 在 SEA 中不输出

**问题**: meow 的 `--version` 通过 `import.meta` 找 package.json，在 SEA 环境中 import.meta.url 是 blob URL，无法解析。

**修复**:
- `scripts/bundle.mjs`: esbuild `define` 注入 `process.env.QINCODE_VERSION` 为打包时的版本号
- `src/index.ts`: 读取 `process.env.QINCODE_VERSION` 传给 meow 的 `version` 选项

#### 4. 改进 SEA wrapper 临时文件清理

- 清理逻辑从只删 .mjs 文件扩展为同时删除 HTML 文件和临时目录

### 验证结果

- `npm run bundle` 成功 → 生成 `release/qincode-darwin-arm64`（228M）
- `qincode --version` → 输出 `0.1.0-build.2740` ✓
- `qincode --help` → 正常输出 ✓
- 临时文件运行后自动清理 ✓
- 带 quarantine 属性的二进制在 terminal 中可正常运行 ✓

### 当前已知限制

1. **SEA 只能构建当前平台**: 目前只生成 darwin-arm64 二进制，darwin-x64 和 linux 版本需要在 GitHub Actions CI 中构建
2. **macOS ad-hoc 签名**: 二进制使用 ad-hoc 签名，spctl 会显示 rejected，但 CLI 工具在 terminal 中直接运行不受 Gatekeeper 影响
3. **文件大小**: 228M（包含完整 Node.js 运行时），这是 SEA 的正常体积
4. **OAuth HTML 文件**: SEA 模式下运行时动态提取到临时目录，退出时自动清理
