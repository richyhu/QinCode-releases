# QinCode 打包分发 — 变更文档

## 第1次变更
- **时间**: 2026-06-22 10:58
- **对话次数**: 1

### 变更内容

#### 1. 打包方案从 @yao-pkg/pkg 切换到 Node.js SEA

**原因**: @yao-pkg/pkg 的远程预编译缓存 404（node18 和 node20 均不可用），导致需要从源码编译 Node.js，耗时 30-60 分钟且磁盘空间不足。

**新方案**: 使用 Node.js 原生 SEA (Single Executable Application) 功能。

#### 2. 修改文件: `scripts/bundle.mjs`

- esbuild 输出格式保持 ESM（.mjs），添加 `createRequire` banner 解决 CJS `require()` 兼容问题
- 创建 CJS wrapper（`qincode-sea-entry.cjs`）作为 SEA 入口，动态 import ESM bundle
- ESM bundle 和 HTML 资源作为 SEA assets 嵌入二进制
- 运行时从 SEA assets 提取 .mjs 到临时目录再执行
- 修复 Node.js v24+ 二进制中 sentinel fuse 重复的问题（patch 掉多余的 fuse）
- macOS 特定处理: codesign --remove-signature → postject 注入 → codesign --sign -
- 只构建当前平台二进制，跨平台构建留给 CI

#### 3. 新增依赖: `react-devtools-core`（devDependency）

ink 的 reconciler.js 在 DEV=true 时动态 import react-devtools-core，之前标记为 external 导致 SEA 运行时找不到。现在移除 external 标记并安装该包。

#### 4. 新增依赖: `postject`（devDependency）

用于将 SEA blob 注入 Node.js 二进制文件。
